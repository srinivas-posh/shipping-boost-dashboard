Exp 165633 — Shipping Time Boost: Impression Shift Analysis

Search ranking experiment: how shipping speed boosts shift impressions across page positions
50,000 impressions · dataw90 · verified ✓

How the Boost Works

Each listing in a search session is assigned a shipping time bucket based on its average shipping speed (D1 = ships in 1 day, D2 = 2 days, …, D7+ = 8+ days, did_not_ship = never shipped, no_sale = no sales history). The experiment adds a boost to the listing's base search score proportional to how fast it ships:

new_score = search_listing_score + boost[shipping_bucket] × importance
// importance is a global weight (held constant across variants)
// listings with faster shipping get larger boosts → move to earlier pages

Within each search session (search_page_group_id), all listings are re-ranked by their new scores. Page bucket is determined by rank position: P1 = ranks 1–40, P2 = 41–80, …, P7 = 241–280. Because total impressions are conserved, every listing that moves to an earlier page displaces another listing to a later page — so delta column totals always sum to zero.

Two bucketing strategies are compared: Lifetime uses avg_shipping_time_lifetime, and 90-Day uses avg_shipping_time_90d. Same boost values apply to both; only the bucket assignment changes.

Reading These Tables

Each table shows impressions by shipping bucket (rows) × page bucket (columns). A cell value like D1 / P1 = 1,601 means 1,601 D1-speed listings appeared on page 1 under that condition. The Row Total is the total impressions for that shipping bucket across all pages; this is invariant — re-ranking cannot change how many D1 listings there are, only which page they land on. The Column Total is the total impressions on that page across all buckets; this is also invariant (each page always shows 40 results × number of sessions).

Absolute Distributions — Variant 1 (Moderate Boost) V1

What V1 Does

V1 applies a graduated boost from D1 (highest, +1.00) down to D5 (+0.30) and also boosts no_sale listings (+0.30). The net effect: D1–D3 fast shippers rise to earlier pages; slow shippers (D7, D7+, did_not_ship) are unaffected. The no_sale boost is notable — these listings have no shipping history, so V1 treats them charitably.

Row totals are identical to control for each bucket — the same listings exist, they just land on different pages. Column totals (per page) are also identical.

Absolute Distributions — Variant 2 (Aggressive Fast-Ship Boost) V2

What V2 Does

V2 applies a stronger, narrower boost: D1 (+2.00), D2 (+1.50), D3 (+1.00) — and zero for everything else. Unlike V1, D4–D5 and no_sale get no boost. This creates a sharper bifurcation: the fastest third of shippers see large gains, while everyone else is pushed further back.

The effect is particularly visible in D4 and no_sale rows: with no boost in V2, they get crowded to later pages by the strongly-boosted D1–D3 cohort. Compare the D4 P1 cell: 692 (ctrl) → 115 (V1) → 120 (V2) in the Lifetime view. D4 gets hurt in both variants, but V2's higher D1–D3 boost compresses everyone below even harder.

Delta Views — Impression Shift vs Control ±Δ

How to Read Delta Tables

Delta = Variant − Control. A green positive value means that shipping bucket gained impressions on that page relative to control. A red negative value means it lost impressions. Every column total must equal zero — impressions are conserved within each page bucket. Listings that gain P1 impressions must displace an equal number of other listings out of P1.

The intensity of the color scales with magnitude: darker = larger absolute shift. This makes it easy to spot where the biggest movements are (typically D1–D3 gaining at P1, D4–D7+ and no_sale losing at P1).
